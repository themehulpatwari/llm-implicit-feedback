# PAIRWISE_IMPORTANT_FEATURES Documentation

Features used in pairwise preference prediction (target: `binary_preference`). In pairwise tasks the user sees two LLM responses side-by-side (left / right, also called response A / response B) and picks which they prefer.

---

## 1. Metadata / Control Features

| Feature | Description |
|---|---|
| `adjustment` | Per-query adjustment value from the query logs (e.g. difficulty or quality calibration). |
| `llm_1_<model>` | One-hot indicator: response 1 (left) was generated by `<model>`. Models: `claude-3-5-sonnet-20240620`, `claude-sonnet-4-5-20250929`, `deepseek-ai/DeepSeek-V3`, `gpt-4o-mini`, `meta-llama/Llama-3.3-70B-Instruct-Turbo`. |
| `llm_2_<model>` | One-hot indicator: response 2 (right) was generated by `<model>`. Same model set as above. |
| `max_idx_left` | Maximum character index of the left response (proxy for response length). |
| `max_idx_right` | Maximum character index of the right response. |
| `query_length_right` | Length of the query prompt in characters (right-side version). |
| `response_left` | Character length of the left LLM response. |
| `response_right` | Character length of the right LLM response. |
| `total_entries_left` | Total number of tracking data points (gaze + mouse samples) recorded while reviewing the left response. |
| `total_entries_right` | Total number of tracking data points recorded while reviewing the right response. |

---

## 2. Cross-Modality Feature

| Feature | Description |
|---|---|
| `cross_modality_left_reviewing_duration_ratio_gaze_mouse` | Ratio of gaze engaged time to mouse engaged time on the **left** response during reviewing: `gaze_left_reviewing_engaged_time_s / (mouse_left_reviewing_engaged_time_s + 0.001)`. Values > 1 mean the user's gaze was active longer than the mouse. |

---

## 3. Gaze Features — Comparison Level

These capture how differently the user looked at the two responses.

| Feature | Description |
|---|---|
| `gaze_comparison_reviewing_time_ratio` | Ratio of total gaze-on-screen time for left vs. right: `min(gaze_left_reviewing_engaged_time_s / (gaze_right_reviewing_engaged_time_s + 0.001), 100.0)`. Values > 1 mean the user spent more gaze time on the left response. |
| `gaze_comparison_reviewing_activity_diff` | Difference in gaze active ratios between left and right responses: `gaze_left_reviewing_active_ratio − gaze_right_reviewing_active_ratio`. |
| `gaze_left_reviewing_offscreen_ratio` | Mean of `is_not_looking` samples while reviewing the **left** response. Higher = more time looking away from screen. |
| `gaze_left_reviewing_engaged_time_s` | Total seconds the user's gaze was on-screen while reviewing the **left** response. |
| `gaze_right_reviewing_active_ratio` | Gaze engaged time on the **right** response divided by total global reviewing duration (capped at 100). |
| `gaze_right_reviewing_offscreen_ratio` | Mean of `is_not_looking` samples while reviewing the **right** response. |

---

## 4. Mouse Features — Comparison Level

| Feature | Description |
|---|---|
| `mouse_comparison_reviewing_activity_ratio` | Ratio of mouse active ratio on left vs. right: `min(mouse_left_reviewing_active_ratio / (mouse_right_reviewing_active_ratio + 0.001), 100.0)`. Values > 1 mean the mouse was more active while reading the left response. |
| `mouse_comparison_reviewing_time_ratio` | Ratio of mouse engaged time on left vs. right: `min(mouse_left_reviewing_engaged_time_s / (mouse_right_reviewing_engaged_time_s + 0.001), 100.0)`. |
| `mouse_left_reviewing_active_ratio` | Mouse engaged time on the **left** response divided by total global reviewing duration (capped at 100). |
| `mouse_right_reviewing_offscreen_ratio` | Proportion of mouse samples where the mouse was inactive while reviewing the **right** response. |

---

## 5. Gaze Features — Response A (Left)

Per-response gaze features computed for response A only.

| Feature | Description |
|---|---|
| `response_A_gaze_focused_engagement_ratio` | Active engagement ratio: consecutive gaze intervals capped at 2000 ms (to remove long pauses), summed, then divided by the total time span from first to last gaze timestamp. Range 0–1. |
| `response_A_gaze_overall_attention_ratio` | Proportion of response-A gaze samples where the user is actively looking at text (valid coordinates, not offscreen): `len(looking_samples) / len(all_samples)`. |
| `response_A_gaze_window_006` | Gaze density in the 7th character window of response A (response divided into 100 equal-length windows, 0-indexed). |
| `response_A_gaze_window_007` | Gaze density in the 8th character window of response A. |
| `response_A_gaze_window_008` | Gaze density in the 9th character window of response A. |

---

## 6. Mouse Features — Response A (Left)

| Feature | Description |
|---|---|
| `response_A_mouse_focused_engagement_ratio` | Same computation as the gaze version but using mouse activity samples: capped-interval sum / total time span. Range 0–1. |
| `response_A_mouse_normalized_char_position_variance` | Variance of the normalized character positions the user's mouse visited in response A. High variance indicates the mouse moved across a wide span of the response. |
| `response_A_mouse_overall_attention_ratio` | Proportion of response-A mouse samples with valid, active coordinates (not offscreen, not at −1). |
| `response_A_mouse_window_000` | Mouse activity density in the **first** character window (beginning) of response A. |
| `response_A_mouse_window_099` | Mouse activity density in the **last** character window (end) of response A. |
| `response_A_mouse_reading_completion_ratio` | How far through response A the mouse reached: `max_char_position_reached / response_length`. Range 0–1. |

---

## 7. Gaze Features — Response B (Right)

| Feature | Description |
|---|---|
| `response_B_gaze_focused_engagement_ratio` | Same as response A version: active-gaze engagement ratio for response B. |
| `response_B_gaze_normalized_avg_char_position` | Average normalized character position (0–1) gazed at while reading response B. Values near 0.5 suggest the user read through the middle of the response. |
| `response_B_gaze_overall_attention_ratio` | Proportion of response-B gaze samples with active attention (valid coordinates). |
| `response_B_gaze_window_000` | Gaze density in the **first** character window of response B. |
| `response_B_gaze_window_006` | Gaze density in the 7th character window of response B. |
| `response_B_gaze_window_014` | Gaze density in the 15th character window of response B. |

---

## 8. Mouse Features — Response B (Right)

| Feature | Description |
|---|---|
| `response_B_mouse_data_points` | Total number of mouse tracking samples recorded while reviewing response B. |
| `response_B_mouse_focused_engagement_ratio` | Active mouse engagement ratio for response B (capped-interval method, same as response A). |
| `response_B_mouse_normalized_avg_char_position` | Average normalized character position where the mouse was active in response B. |
| `response_B_mouse_overall_attention_ratio` | Proportion of response-B mouse samples with valid, active coordinates. |
| `response_B_mouse_window_022` | Mouse activity density in the 23rd character window of response B. |
| `response_B_mouse_window_031` | Mouse activity density in the 32nd character window of response B. |
| `response_B_mouse_window_033` | Mouse activity density in the 34th character window of response B. |
| `response_B_mouse_window_037` | Mouse activity density in the 38th character window of response B. |
| `response_B_mouse_window_039` | Mouse activity density in the 40th character window of response B. |
| `response_B_mouse_reading_completion_ratio` | How far through response B the mouse reached: `max_char_position_reached / response_length`. Range 0–1. |

---

## Notes

- **Response A / left** and **Response B / right** refer to the same physical response — whichever response appeared on the left side of the comparison screen is response A / left.
- **Window features** (`*_window_NNN`): the response is divided into 100 equal character-length segments. Window 000 is the beginning; window 099 is the end. The value represents the density of gaze/mouse activity in that segment.
- **Active ratio** features are capped at 100 to suppress extreme outliers caused by very short denominators.
- **Offscreen** means the gaze tracker or mouse position recorded data outside the response area (e.g. user looked away or moved mouse off-screen).
- **Normalized character position**: character index divided by response length, giving a 0–1 scale regardless of response length.
